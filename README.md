# CAMPLUS PADEL CUP 2026 — Bracket Generator

A Java command-line tool for managing the knockout stage of the **Camplus Padel Cup 2026** (men's tournament). Starting from group standings and playoff results, it automatically generates overall rankings, round-of-16 matchups, and full brackets for the **Gold** and **Silver** tournaments.

## Features

- **Group standings** with automatic sorting and full tiebreaker resolution (5 criteria)
- **Overall rankings** by position (1st, 2nd, 3rd, and 4th place finishers)
- **Playoff matchups** — 2nd place vs 3rd place (regulation art. 9)
- **Gold round of 16** — 1st place finishers vs playoff winners (art. 10)
- **Silver round of 16** — 4th place finishers vs playoff losers (art. 11)
- **Visual bracket** with a full tree from round of 16 through the final
- **Tiebreaker persistence** — interactive answers are saved to the input file so they don't need to be re-entered on subsequent runs

## Requirements

- Java 17 or later (uses `record`, pattern switch, `String.repeat`)

## Build and run

```bash
javac TorneoPadel.java
java TorneoPadel tournament.txt
```

## Input file format

The file is a plain text file with three sections: **GIRONE** (group), **SPAREGGI** (playoffs), and **TIEBREAK** (auto-generated by the program).

```text
# Lines starting with # are comments and are ignored.
# Blank lines are ignored.

GIRONE 1
Team Alpha ; 3 ; +2 ; +8
Team Bravo ; 2 ; +1 ; +3
Team Charlie ; 1 ; -1 ; -4
Team Delta ; 0 ; -2 ; -7

GIRONE 2
...

GIRONE 8
...

SPAREGGI
1 2 1 1 2 1 2 1
```

### Field reference

**Team row:** `name ; wins ; set_difference ; game_difference`

| Field | Type | Example | Description |
|-------|------|---------|-------------|
| Name | text | `Team Alpha` | Team name |
| Wins | integer | `3` | Matches won in the group stage |
| Set diff. | signed integer | `+2` | Sets won minus sets lost |
| Game diff. | signed integer | `-3` | Games won minus games lost |

**Playoffs row:** 8 space-separated numbers, one per playoff match. `1` = the 2nd-place team wins, `2` = the 3rd-place team wins.

### TIEBREAK section (auto-generated)

When the program interactively asks who won a head-to-head or a coin toss, the answers are automatically appended to the end of the file:

```text
TIEBREAK
G1_SD_Team_Alpha_vs_Team_Bravo ; 1
G3_MON_1_Team_X_Team_Y ; 2
```

On subsequent runs, the program reads these answers instead of asking again.

## Tiebreaker criteria

When two or more teams in a group are tied on wins, the following criteria are applied in order:

1. **Wins** — already present in the file
2. **Head-to-head** — asked interactively, then saved
3. **Set difference**
4. **Game difference**
5. **Coin toss** — asked interactively, then saved

## Code structure

| Class / inner class | Responsibility |
|----------------------|----------------|
| `Squadra` | Team data model |
| `Partita` | Record representing a matchup (team A vs team B) |
| `DatiTorneo` | Container for parsed data (groups + playoff results) |
| `TieBreakStore` | Storage and persistence of tiebreaker answers |
| `ConsoleIO` | Safe terminal input reading |
| `FileParser` | Input file parsing |
| `FileWriter` | Writing tiebreaker data back to the file |
| `TieBreakResolver` | Complete tiebreaker resolution logic |
| `Classifiche` | Overall ranking computation by position |
| `Printer` | Console formatting and output (standings, brackets, trees) |
